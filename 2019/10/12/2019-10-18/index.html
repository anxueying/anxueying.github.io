Quick Start

Algorithm

一个leetcode算法题

链表的特点：

  • 不需要连续的内存空间

  • 链表中每个节点的结构：

    • data：数据
    • next：后继指针，下一个节点的地址
    • prev：双向链表特有

  • 头结点：第一个结点，记录链表的基地址

  • 尾结点：最后一个结点

  • 支持查找O(n)、插入O(1)、删除O(1)操作

常见类型

  1. 单链表

    • 节省内存

  2. 循环链表

    • 链尾到链头实现更容易
  3. 双向链表（空间换时间）
    • 插入和删除给定指针指向的节点更效率O(1)
    • 按值查询的效率更高一些，是单链的一半

  4. 双向循环链表

  5. 删除链表中的节点（leetcode237）

    从链表中删除一个数据有两种情况：

    • 删除节点中“值等于某个给定值”的节点（本题）
      • 无论单链还是双链，查找的时间复杂度都为O(n)
      • 双链效率更高一点
    • 删除给定指针指向的节点
      • 双链表是O(1)，单链表是O(n)
写链表代码的技巧：

  1. 理解指针或引用的含义

    1. p->next=q p 结点中的 next 指针存储了 q 结点的内存地址。
    2. p->next=p->next->next p 结点的 next 指针存储了 p 结点的下下一个结点的内存地址。

  2. 警惕指针丢失和内存泄漏

    • 插入节点时注意操作顺序
    • 删除节点时记得手动释放内存空间（java不需要）
